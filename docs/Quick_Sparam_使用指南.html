绘图或分析之前先修正端口信息，可以避免后续端口误配。

1

3

工作流：DDR 端到端串扰与信道分析

DDR5 / LPDDR5X 通道从封装+PCB+封装 S 参数到时域 SNR 的完整分析流程。
应用场景：DDR5 / LPDDR5X 信号完整性分析，从三段 S 参数到时域 SNR 评估
涉及模块：S参数级联 → 差分转换（部分） → 端口管理（阻抗归一化） → 频域分析 → 时域分析
典型输入件：SoC_pkg.snp  +  PCB_board.snp  +  DRAM_pkg.snp
1

S 参数级联

可用 → 模块：S参数级联
将 SoC PKG、PCB Board 和 DRAM PKG 三段 S 参数依次级联，生成端到端通道模型。级联顺序固定：SoC_pkg.snp → board.snp → dram_pkg.snp。每段文件的端口顺序应事先确认（驱动侧端口在左，负载侧端口在右）。
DDR5/LPDDR5 通常为 50 Ω 单端参考阻抗；确保三段文件参考阻抗一致，否则级联前需先做阻抗归一化。
2

部分差分转换

可用 → 模块：差分转换
仅对 DQS 和 WCK 差分时钟线做单端→差分转换，其余 DQ 数据线保持单端。在对话框中选「部分差分」，在 line 输入框填写 DQS/WCK 对应的 line 编号。
DDR DQS 为差分读写选通信号，LPDDR5 WCK 为写时钟；DQ 为单端数据信号，不需要转换。差分阻抗通常设 100 Ω。
3

端口阻抗归一化

可用 → 模块：端口管理
根据 SoC 驱动阻抗 Ron 和 DRAM 片上终端 ODT 修正参考阻抗，使 S 参数反映实际电路工作点。在「端口管理 → 阻抗变换」中设置驱动端和接收端端口的参考阻抗。
DDR5 典型值：Ron ≈ 34 Ω，ODT ≈ 40~48 Ω；LPDDR5X 可能更低（Ron ≈ 20~30 Ω）。需查阅 PHY Databook 和 DRAM 规格书。
4

频域批量分析

可用 → 模块：频域分析
对端到端通道做频域分析：重点关注串扰和（NEXT + FEXT 之和）、VTF（电压传输函数，即 SDD21）以及插损在 DDR5/LPDDR5X 奈奎斯特频率处的数值。勾选「最差 line」模式可快速定位问题通道。
DDR5-6400 奈奎斯特 = 3.2 GHz；LPDDR5X-8533 奈奎斯特 ≈ 4.27 GHz；LPDDR5X-9600 奈奎斯特 ≈ 4.8 GHz。
5

时域波形及 SNR 分析

开发中 → 模块：时域分析
将 S 参数转换到时域，观察脉冲响应，叠加串扰后评估 ISI + 串扰对 SNR 的影响。
该功能正在开发中，预计支持：脉冲响应叠加、SNR 自动计算、眼图前分析（Eye Diagram pre-cursor analysis）。
当前可使用「时域分析」模块查看 TDR / 冲激 / 阶跃响应作为初步参考；完整 SNR 分析模块上线后将直接集成到此工作流。
